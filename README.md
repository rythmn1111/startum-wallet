## 🌟 What is Startum Wallet?

Startum Wallet turns a plain NFC card into a **self-custodial crypto wallet** you can tap anywhere to pay — like a contactless bank card, but with no bank, no seed phrase memorisation, and no single point of failure.

Under the hood, your private key is **split across two places**: your physical NFC card and a secure server. Neither half alone can sign anything. You need **both** — the card tap + your password — to authorise a transaction. This is powered by **BitGo's battle-tested 2-of-3 multisig infrastructure**, the same technology used by institutional crypto custodians.

### 🔒 Split-Key Architecture
Your private key is XOR-split: one half written to the NFC card, one half encrypted on the server. **Neither half is ever stored whole anywhere.** The key only exists in memory for milliseconds during a transaction.

</td>
<td>

### 🕵️ Fresh Address Privacy
Every payment you receive goes to a **new on-chain address**, generated by BitGo's HD wallet. Your payment history is unlinkable — no one can track your income by watching a single address.

</td>
</tr>
<tr>
<td>

### 🔷 ENS Programmable Payment Rules
Set your **ENS text records** to configure how payments behave:
- Cap incoming amounts (`nfcwallet.max-amount`)
- Auto-split to savings (`nfcwallet.auto-split: savings.eth:20`)
- Require NFC card presence for payments
- Receive any token as ETH (`nfcwallet.receive-as: eth`)

</td>
<td>

### 🧾 Encrypted Receipts via Fileverse
Every payment you collect creates an **end-to-end encrypted document** on Fileverse, anchored on-chain. No Stripe. No bank. Just a permanent, private receipt owned only by you — accessible at `ddocs.new`.

</td>
</tr>

### 🛡️ Selective Disclosure Proof
When paying, the app generates a **cryptographic proof** (ECDSA over a structured message) that proves:
- ✅ You control the wallet
- ✅ You authorise this exact amount to this exact receiver
- ❌ Without revealing your balance, history, or identity

</td>

Onboarding takes 60 seconds. Tap your NFC card, set a password — that's it. No 24-word backup to write down or lose. Recovery is handled by the split-key scheme.

## 🎯 How a Payment Works

```
MERCHANT SIDE                           PAYER SIDE
─────────────                           ──────────
1. Open Pay tab
2. Enter amount + chain
   (ENS rules auto-applied)

3. Tap "Collect"
   ──────── NFC scan ────────►  Payer taps their NFC card
                                Card transmits: walletId, nfcHalf

4. App reconstructs key:
   XOR(nfcHalf, serverHalf) → AES-GCM ciphertext
   AES-GCM decrypt → private key (in memory, ~50ms)

5. 🛡️ Selective Disclosure Proof generated
   (proves capacity without revealing balance)

6. BitGo executes transfer:
   Fresh receive address ← ETH ← Payer's BitGo wallet

7. 🧾 Fileverse receipt created (fire-and-forget)
   E2E encrypted, on-chain, accessible at ddocs.new

8. Key wiped from memory ✓
```

## 🚀 Getting Started

### Prerequisites
- Node.js 18+
- React Native environment (Xcode for iOS, Android Studio for Android)
- PostgreSQL
- BitGo testnet account → [app.bitgo-test.com](https://app.bitgo-test.com)

### Run the backend
```bash
cd backend
cp .env.example .env          # fill in BITGO_ACCESS_TOKEN, JWT_SECRET
npm install
npm run migrate               # sets up PostgreSQL tables
npm start
```

### Run the app
```bash
cd app
npm install
npx react-native run-ios      # or run-android
```

### Program your NFC card
1. Create a wallet in the app
2. Go to the **Card** tab
3. Tap "Write to Card" and hold your NFC card to the phone

## 🔐 Security Model

| Threat | Protection |
|---|---|
| Phone stolen | NFC card required — phone alone is useless |
| Card stolen | Password required — card alone is useless |
| Server breached | Server only holds encrypted half — no key material exposed |
| Replay attacks | Selective disclosure proof expires after 5 minutes |
| Address tracking | Fresh address per payment — on-chain privacy |
| Receipt privacy | Fileverse E2E encryption — no third party can read receipts |
